# Docker Homework

این مخزن شامل جواب تمرین‌های Docker است. جواب‌ها بر اساس شماره تمرین‌ها در پوشه‌های جداگانه قرار گرفته‌اند تا بررسی و اجرای هر بخش ساده‌تر باشد.

## ساختار کلی
```text
docker_homework/
├── 01_docker_images/
├── 02_containers_1/
├── 03_containers_2/
├── 04_volumes_network_1/
├── 05_volumes_network_2/
├── 06_docker_compose/
├── 07_sample_project/
├── README.md
└── tree.txt
```
## نحوه قرارگیری جواب‌ها

### تمرین‌های 01 تا 05

برای تمرین‌های `01` تا `05`، جواب هر سوال در پوشه مربوط به همان سوال قرار دارد.

برای مثال، جواب سوال `1.2` در مسیر زیر قرار گرفته است:

text
01_docker_images/1.2/

نمونه ساختار:

```text
01_docker_images/
├── 1.1/
├── 1.2/
└── 1.3/

02_containers_1/
├── 2.1/
├── 2.2/
├── 2.3/
└── 2.4/
```

در هر پوشه، فایل‌های مربوط به همان سوال قرار داده شده‌اند؛ مثل اسکریپت‌ها، خروجی دستورات، توضیحات و فایل‌های موردنیاز برای اجرای تمرین.

### تمرین‌های 06 و 07

برای تمرین‌های `06` و `07`، جواب‌ها به صورت زیرپوشه برای هر سوال جدا نشده‌اند.

در این دو بخش، فایل‌های پاسخ مستقیما داخل پوشه اصلی همان تمرین قرار دارند:

```text
06_docker_compose/
07_sample_project/
```
دلیل این ساختار این است که تمرین‌های 06 و 07 پروژه‌محور هستند و فایل‌ها به صورت یک پروژه کامل کنار هم قرار گرفته‌اند.

## توضیح پوشه‌ها

### `01_docker_images/`

این پوشه شامل جواب تمرین‌های مربوط به Docker Image است.

موضوعات این بخش شامل مواردی مثل نصب Docker، بررسی نسخه Docker، عملیات روی imageها، بررسی layerها و history image است.

ساختار کلی:

```text
01_docker_images/
├── 1.1/
├── 1.2/
└── 1.3/
```
### `02_containers_1/`

این پوشه شامل جواب تمرین‌های اولیه مربوط به containerها است.

موضوعات این بخش شامل اجرای container، بررسی وضعیت container، متغیرهای محیطی، port mapping و اجرای دستور داخل container است.

ساختار کلی:

```text
02_containers_1/
├── 2.1/
├── 2.2/
├── 2.3/
└── 2.4/
```
### `03_containers_2/`

این پوشه ادامه تمرین‌های مربوط به containerها را شامل می‌شود.

موضوعات این بخش شامل logها، restart policy، محدودیت منابع و health check است.

ساختار کلی:

```text
03_containers_2/
├── 3.1/
├── 3.2/
├── 3.3/
└── 3.4/
```
### `04_volumes_network_1/`

این پوشه شامل تمرین‌های مربوط به volume و bind mount است.

موضوعات این بخش شامل ساخت volume، استفاده از bind mount و backup گرفتن از volume است.

ساختار کلی:

```text
04_volumes_network_1/
├── 4.1/
├── 4.2/
└── 4.3/
```
### `05_volumes_network_2/`

این پوشه شامل تمرین‌های مربوط به network در Docker است.

موضوعات این بخش شامل ساخت network، ارتباط containerها و بررسی isolation بین networkها است.

ساختار کلی:

```text
05_volumes_network_2/
├── 5.1/
├── 5.2/
└── 5.3/
```
### `06_docker_compose/`

این پوشه شامل تمرین Docker Compose است.

در این بخش فایل‌ها به صورت پروژه‌ای قرار گرفته‌اند و برای هر سوال، پوشه جداگانه ساخته نشده است.

فایل‌های مهم این بخش:

```text
06_docker_compose/
├── app/
├── web/
├── docker-compose.yml
├── .env
├── compose_operations.sh
├── compose_status.txt
├── compose_summary.txt
└── service_communication.txt
```
### `07_sample_project/`

این پوشه شامل پروژه نمونه نهایی است.

در این بخش نیز جواب‌ها به صورت پوشه جدا برای هر سوال قرار ندارند و همه فایل‌های پروژه در یک ساختار واحد قرار گرفته‌اند.

فایل‌های مهم این بخش:

```text
07_sample_project/
├── app/
├── nginx/
├── web/
├── docker-compose.yml
├── run_project.sh
├── project_logs.txt
├── project_status.txt
└── README.md
```
## انواع فایل‌ها

در پوشه‌های تمرین ممکن است فایل‌های زیر وجود داشته باشد:

- فایل‌های `.sh`: اسکریپت‌های اجرای دستورات Docker
- فایل‌های `.txt`: خروجی دستورات یا توضیحات تمرین
- فایل‌های `.yml`: فایل‌های Docker Compose
- فایل‌های `Dockerfile`: فایل ساخت image
- فایل‌های برنامه مثل `.py`، `.html` و تنظیمات پروژه

## نحوه بررسی تمرین‌ها

برای بررسی جواب هر سوال:

1. وارد پوشه مربوط به شماره تمرین شوید.
2. اگر تمرین بین `01` تا `05` است، وارد پوشه شماره سوال شوید.
3. فایل‌های `.sh` شامل دستورات اجرا شده هستند.
4. فایل‌های `.txt` شامل خروجی دستورات یا توضیحات مربوط به همان سوال هستند.
5. برای تمرین‌های `06` و `07`، فایل‌های اصلی پروژه مستقیما داخل پوشه همان تمرین قرار دارند.

مثال:

bash
cd docker_homework/01_docker_images/1.2

یا برای تمرین Docker Compose:

bash
cd docker_homework/06_docker_compose

## خلاصه ساختار پاسخ‌دهی

- تمرین‌های `01` تا `05`: هر سوال در پوشه جداگانه خودش قرار دارد.
- تمرین‌های `06` و `07`: جواب‌ها به صورت پروژه‌ای و در همان پوشه اصلی تمرین قرار دارند.
- فایل‌های خروجی و اسکریپت‌ها کنار هم قرار گرفته‌اند تا روند انجام تمرین قابل بررسی باشد.
