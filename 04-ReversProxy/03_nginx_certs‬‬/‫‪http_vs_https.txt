1
HTTP: داده‌ها را رمزنگاری نمی‌کند؛ اطلاعات به صورت متن ساده منتقل می‌شوند و ممکن است در مسیر شنود شوند.
HTTPS: نسخه امن HTTP است که با رمزنگاری SSL/TLS داده‌ها را محافظت می‌کند، بنابراین اطلاعات بین کاربر و سرور امن و غیرقابل خواندن برای دیگران منتقل می‌شود.

2
در Docker باید برای HTTP و HTTPS پورت‌های متفاوت استفاده شود، چون این دو پروتکل 
سرویس‌های جداگانه با رفتار متفاوت هستند. HTTP معمولاً روی پورت 80 کار می‌کند و داده‌ها را 
بدون رمزنگاری منتقل می‌کند، در حالی که HTTPS روی پورت 443 اجرا می‌شود و با استفاده از
TLS/SSL و گواهی امنیتی ارتباط را رمزنگاری می‌کند. به همین دلیل در Docker لازم است پورت‌های 
کانتینر به پورت‌های میزبان map شوند تا مشخص باشد درخواست‌های معمولی (HTTP) و 
درخواست‌های امن (HTTPS) از چه پورتی وارد کانتینر می‌شوند

3
HTTP و HTTPS را می‌توان در یک container فعال کرد با این کار که وب‌سرور (مثل Nginx) را طوری 
پیکربندی کنیم که روی دو پورت مختلف گوش دهد؛ معمولاً پورت 80 برای HTTP و پورت 443 برای
HTTPS. سپس هنگام اجرای container هر دو پورت را به میزبان map می‌کنیم (مثلاً -p 80:80 و -p
443:443). همچنین برای HTTPS باید گواهی SSL/TLS در تنظیمات سرور قرار داده شود تا ارتباط
امن برقرار شود.
